# 工作备忘录 使用说明书

> 版本：v2.4.0 ｜ 作者：梁兴宇 ｜ 适用平台：iOS / Android

---

## 一、功能概览

工作备忘录是一款基于 PWA 技术的移动端工作管理工具，添加到手机桌面后如同原生 APP 一样使用，数据全部保存在手机本地，无需注册登录。

| 功能 | 说明 |
|------|------|
| 📝 工作事项 | 创建、分类、设置优先级和截止日期，进度条直观展示，逾期自动标记 |
| 🎤 语音输入 | 语音添加事项（自动识别日期和分类），语音更新进度 |
| 📊 进度追踪 | 百分比制进度（0%-100%），100% 自动标记完成，月历视图 |
| 📈 周报月报 | 智能生成报告（概览统计+新增/未完成/逾期/下月计划），导出 Excel 和微信分享 |
| 🔔 每日提醒 | 每天 17:00（可自定义）提醒未完成事项 |
| 💬 微信推送 | 自动将未完成事项推送到微信（PushPlus / WxPusher 免费无限） |
| ♻️ 回收站 | 删除的事项 5 天内可恢复 |
| 💾 自动备份 | 每日自动全量备份，保留 3-10 份，可随时恢复 |
| 📤 数据管理 | JSON 备份导入/导出、可读文本导出 |
| 🔗 分享应用 | 一键分享给同事朋友使用 |

## 二、系统要求

### 硬件要求

| 项目 | 最低要求 |
|------|---------|
| 手机 | iPhone 8 及以上 / 安卓 8.0 及以上 |
| 存储空间 | 约 2MB（应用本身极小，数据占用取决于使用量） |

### 软件要求

| 平台 | 浏览器 | 版本要求 |
|------|--------|---------|
| iOS | Safari | 14.4 及以上 |
| Android | Chrome | 80 及以上 |

> ⚠️ 不建议使用微信内置浏览器、华为浏览器、UC浏览器等第三方浏览器，功能支持不完整。

### 网络要求

- **首次打开**：需要网络连接加载页面
- **添加到桌面后**：离线也可正常使用（自动缓存）
- **微信推送功能**：需要网络连接
- **语音识别**：需要网络连接（在线语音识别）

## 三、安装步骤

### iPhone 安装

1. 打开 **Safari 浏览器**（必须是 Safari，不能是其他浏览器）
2. 地址栏输入：`https://lxy505.github.io/work-memo/`
3. 等待页面完全加载
4. 点击底部 **分享按钮**（↑ 方形向上箭头图标）
5. 在弹出菜单中向下滑动，找到 **"添加到主屏幕"**
6. 点击 **"添加"** 确认
7. 返回桌面，看到"备忘录"图标即安装成功

### Android 安装

1. 打开 **Chrome 浏览器**
2. 地址栏输入：`https://lxy505.github.io/work-memo/`
3. 等待页面完全加载
4. 点击右上角 **⋮** 菜单
5. 选择 **"添加到主屏幕"** 或 **"安装应用"**
6. 点击 **"添加"** 确认
7. 返回桌面，看到图标即安装成功

### 安装验证

点击桌面图标打开应用，如果出现全屏界面（没有浏览器地址栏），说明安装正确。

## 四、使用说明

### 4.1 工作事项

#### 添加事项

- 点击右上角 **+** 按钮
- 填写事项标题（必填）、描述（选填）
- 选择分类和优先级
- 设置截止日期（可选）
- 拖动滑块设置初始进度（默认 0%）
- 点击 **"添加"**

#### 编辑事项

- **点击事项标题文字**即可打开编辑弹窗
- 支持修改：标题、描述、分类、优先级、截止日期

#### 语音添加事项

- 点击右上角 **🎤** 按钮
- 点击 **"开始录音"**
- 说出事项名称，系统自动识别日期、截止日期和分类
- 示例：

| 语音输入 | 解析结果 |
|---------|---------|
| "下周四方案汇报" | 标题：方案汇报 ｜ 日期：下周四 ｜ 分类：会议沟通 |
| "5月15日前完成方案设计" | 标题：方案设计 ｜ 截止：5月15日 ｜ 分类：文档撰写 |
| "排查线上故障" | 标题：排查线上故障 ｜ 日期：今天 ｜ 分类：问题处理 |
| "买咖啡" | 标题：买咖啡 ｜ 日期：今天 ｜ 分类：其他 |

#### 语音更新进度

- 点击 **🎤** 按钮开始录音
- 说出格式：**"事项关键词 + 完成 + 百分比"**

| 语音输入 | 效果 |
|---------|------|
| "中矿集团方案完成100%" | 搜索"中矿集团方案"事项，进度设为 100% |
| "登录页面完成了50%" | 搜索"登录页面"事项，进度设为 50% |

> 系统会自动搜索匹配的事项。如果匹配到多个，会让你选择；如果没有匹配，会询问是否新建。

#### 语音日期识别

语音中支持以下日期表述：

| 日期表述 | 说明 |
|---------|------|
| "周一线下会议" | 记录到本周一 |
| "下周三完成方案" | 记录到下周三 |
| "5月20日开会" | 记录到5月20日 |
| "3号前完成" | 截止日期设为本月3号 |
| "下周四前完成" | 截止日期设为下周四 |
| 无日期关键字 | 默认为当天 |

#### 更新进度

- 方式一：点击事项卡片上的 **进度条**
- 方式二：点击事项卡片上的 **"📊 更新进度"** 按钮
- 方式三：切换到"进度"页面，点击事项卡片
- 拖动滑块调整百分比，100% 自动标记为已完成

#### 筛选事项

- 点击顶部的 **"全部"/"进行中"/"已完成"** 切换查看
- 点击分类标签（如"项目开发""会议沟通"）筛选特定分类

#### 逾期标记

- 设置了截止日期且已过期但未完成的事项自动标记为 **"逾期"**
- 逾期事项显示红色标签，排序靠前

#### 删除事项

- 点击事项卡片右上角 **×** 按钮
- 删除后进入回收站，5 天内可恢复

---

### 4.2 工作进度

- 切换到 **"进度"** 页面查看月历视图
- **月历**：显示当月完整日历，有事项的日期标绿点，今天高亮
- 左右箭头切换月份
- 点击日期选中
- 下方显示所有未完成（进度 < 100%）的事项列表
- 每个事项显示：标题、进度条、最新进度说明、截止日期、分类
- 逾期事项红色标记，排序靠前
- 点击事项卡片可直接更新进度

---

### 4.3 周报月报

#### 生成报告

1. 切换到 **"报告"** 页面
2. 选择 **"周报"** 或 **"月报"**
3. 选择对应的周次或月份
4. 点击 **"生成"**

#### 报告格式

生成后的报告包含以下内容：

**概览统计：**

> 本周/月共计 XX 个事项，其中完成 XX 个、新增 XX 个、未完成 XX 个、逾期 XX 个

**1、本周/月新增（XX个）：** 列出该时段内新创建的事项

**2、本周/月未完结事项（XX个）：** 列出进度未达100%的事项

**3、本周/月逾期事项（XX个）：** 列出已过截止日期但未完成的事项

**4、下月已确定工作事项（XX个）：** 列出截止日期在下个月的未完成事项

#### 导出 Excel

- 生成报告后，点击 **"📊 导出Excel"**
- 下载 CSV 文件，可用 Excel/WPS 打开

#### 发送微信

- 点击 **"💬 发送微信"**
- 调用系统分享功能，可选择微信好友或群聊发送

---

### 4.4 每日提醒

- 默认每天 **17:00** 提醒未完成事项
- 可在设置中修改提醒时间
- 可开关提醒功能
- 提醒方式：应用内弹窗 + 浏览器通知（需授权）

---

### 4.5 微信推送

将每日未完成事项自动推送到微信，推荐使用免费方案：

| 方案 | 免费额度 | 推荐度 | 说明 |
|------|---------|--------|------|
| **PushPlus** | 200条/天 | ⭐⭐⭐ | 最推荐，配置简单 |
| **WxPusher** | 无限条 | ⭐⭐⭐ | 需要创建应用 |
| 企业微信机器人 | 无限条 | ⭐⭐ | 需要企业微信群 |
| Server酱 | 5条/月 | ⭐ | 太少，不推荐 |

#### PushPlus 配置步骤

1. 微信搜索并关注 **"PushPlus推送"** 公众号
2. 访问 [pushplus.plus](https://www.pushplus.plus/) 登录
3. 复制你的 Token
4. 在应用 **设置 → 微信推送 → PushPlus** 中粘贴 Token
5. 点击 **"测试发送"** 验证

#### WxPusher 配置步骤

1. 微信搜索并关注 **"WxPusher"** 公众号
2. 访问 [wxpusher.zjiecode.com](https://wxpusher.zjiecode.com/) 创建应用
3. 获取 AppToken 和你的 UID
4. 在应用设置中填入 Token 和 UID
5. 测试发送验证

### 4.6 自动备份

- 默认每天 **23:00** 自动全量备份
- 备份包含：所有事项、进度记录、设置
- 默认保留最近 **3 份**，可在设置中选择保留 3/5/7/10 份
- 超出份数自动删除最旧的备份
- 设置页可查看备份列表，点击 **"恢复"** 还原任意备份

配置路径：**设置 → 自动备份**

| 设置项 | 选项 |
|-------|------|
| 启用自动备份 | 开/关 |
| 备份时间 | 默认23:00，可选 |
| 保留份数 | 3/5/7/10 份 |

### 4.7 分享应用

- 进入 **设置 → 分享给朋友**
- **"📤 分享应用"**：调用系统分享，可直接发送到微信
- **"📋 复制链接"**：复制网址，粘贴发送给朋友

### 4.8 回收站

- 删除的事项自动进入回收站
- **5 天内**可恢复，超过 5 天自动永久删除
- 进入 **设置 → 回收站** 查看
- 点击 **"恢复"** 还原事项

---

### 4.9 数据管理

#### 导出备份（JSON）

- 导出完整数据文件，包含所有事项、进度记录和设置
- 建议定期导出，保存到微信收藏或网盘
- 文件格式：`.json`

#### 导出可读文本

- 导出格式化的 `.txt` 文本文件
- 可用任意文本编辑器打开阅读
- 包含：工作事项列表、进度记录、统计概览

#### 导入备份

- 选择之前导出的 `.json` 文件
- 导入会覆盖当前数据，请谨慎操作
- 适用于换手机或数据恢复场景

## 六、注意事项

### 数据安全

| ⚠️ 重要事项 | 说明 |
|------------|------|
| 数据存储在浏览器本地 | 不会上传到任何服务器，但清除浏览器数据会丢失 |
| 自动备份 | 建议开启自动备份，默认每天23:00备份，保留3份 |
| 手动备份 | 建议每周导出一次备份（JSON），保存到微信收藏或网盘 |
| 清除缓存风险 | 不要在手机设置中清除 Safari/Chrome 的"网站数据"，否则数据会丢失 |
| 换手机 | 需要先在旧手机导出备份，再在新手机导入 |

### 应用更新

- 代码更新后，重新打开应用会自动加载新版本
- 如果看不到更新，删除桌面图标重新添加即可
- 更新不会影响已有数据

### 使用限制

| 限制 | 说明 |
|------|------|
| 语音识别 | 需要网络连接；Safari 上稳定性不如 Chrome；嘈杂环境影响识别率 |
| 推送通知 | iPhone 需要在设置中允许 Safari 通知；安卓需允许 Chrome 通知 |
| 离线使用 | 添加到桌面后可离线使用，但语音识别和微信推送需要网络 |
| 数据不同步 | 多台设备数据独立存储，不支持自动云同步 |
| 存储容量 | localStorage 上限约 5MB，足够存储数千条记录 |

### GitHub 托管

- 本应用托管在 GitHub Pages 上，完全免费
- 只要你的 GitHub 账号和仓库不被删除，应用就能持续访问
- **请不要删除 `lxy505/work-memo` 仓库**
- 建议每年至少登录一次 GitHub 保持账号活跃

## 七、常见问题

**Q：添加到主屏幕后打开是白屏/404？**

A：删除桌面图标，用浏览器重新打开网址，再次添加到主屏幕。确保网址输入正确。

**Q：语音识别没反应？**

A：1) 确保有网络连接；2) 在浏览器设置中允许麦克风权限；3) 推荐使用 Chrome 浏览器；4) 尝试在安静环境下使用。

**Q：语音识别带"下周四""完成"等词时识别错误？**

A：系统会自动识别日期关键词和"完成"等动词，清理后只保留事项标题。如"下周四完成方案汇报"→ 标题"方案汇报"，日期"下周四"。如果仍不正确，可添加后点击标题编辑修改。

**Q：收不到微信推送？**

A：1) 确认已正确填写 Token/UID；2) 点击"测试发送"验证配置；3) 检查是否已关注对应公众号；4) 确认手机网络正常。

**Q：收不到 17:00 提醒？**

A：1) 确认设置中"启用提醒"已开启；2) 允许浏览器通知权限；3) 应用需要处于打开或后台运行状态；4) iPhone 的 Safari 后台运行受限，建议设置推送通知作为补充。

**Q：数据不小心清空了怎么办？**

A：1) 检查自动备份：设置 → 自动备份 → 查看备份列表 → 恢复；2) 如果之前手动导出过备份，通过"导入备份"恢复；3) 没有任何备份则数据无法恢复。

**Q：可以两台手机同步数据吗？**

A：不支持自动同步。手动方式：A 手机导出备份 → 传给 B 手机 → B 手机导入备份。

**Q：事项分类不对怎么办？**

A：点击事项标题打开编辑弹窗，可手动修改分类。语音录入时系统根据关键词自动匹配，未匹配到的归入"其他"。
